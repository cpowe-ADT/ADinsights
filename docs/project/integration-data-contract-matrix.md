## Meta Organic Content Operations (Foundation)

This row records the intended contract boundary for the Content Operations module. As of
2026-06-10, the backend `content_ops` model foundation, initial migration, tenant-scoped DRF API
skeleton, workflow state actions, module-local role gates, separated readiness endpoint, and
planned API/eval gates exist. Basic report endpoints read only stored aggregate
`OrganicPostMetricSnapshot` rows, the content-plan export returns client-safe JSON snapshots, and
persisted export artifacts support repeat downloads. Due-schedule, retryable-attempt, and
queued-attempt processor scans run every minute on disabled-by-default/fakeable provider
boundaries. `publish-now` remains a `501 not_implemented` stub, while the published-post
metric-refresh action now bridges already-synced Meta post insight rows into aggregate snapshots.
No live Meta publishing calls, live AI provider calls, dbt models, live Instagram Graph adapter, or
OAuth scope changes exist yet; `/content` frontend work remains a partial foundation.

## Open Validation Actions

1. Meta authenticated portal validation remains a manual external step because Meta developer docs are crawler-restricted.
2. GA4 and Search Console are now Phase 2 pilot contracts with raw/staging/mart + API exposure; production rollout still requires real OAuth credentials and external connector validation.
3. Contract drift gate is enforced by:
   - `python3 infrastructure/airbyte/scripts/check_data_contracts.py`
   - `PYTHONPATH=backend:. backend/.venv/bin/python -m pytest backend/tests/test_data_contract_checks.py -q`
4. Airbyte operational health checks now interpret cron schedules for a bounded subset of expressions (`* * * * *`, `*/N * * * *`, `M * * * *`, `M */N * * *`, `M H * * *`, `M H * * DOW`). Unsupported cron patterns intentionally fall back to `AIRBYTE_FALLBACK_STALE_MINUTES`.
