# Decision Checklist (v0.1)

Purpose: decide when to update docs, AGENTS, or the doc index.

## Update AGENTS.md when:

- A new cross-cutting doc is added (feature catalog, ownership, escalations).
- Guardrails or schedules change.
- Testing matrix changes.
- Recontext workflow changes.

## Update docs/ops/doc-index.md when:

- A new doc/runbook/checklist is added.
- A doc is moved or deprecated.

## Update feature catalog when:

- A feature changes status (Built/In Progress/Planned).
- Scope changes in a workstream.

## Update API contract changelog when:

- API payloads or schemas change.
- A field is added/removed/renamed.

## Update runbooks when:

- Operational behavior changes.
- Observability/alerts are added or modified.
